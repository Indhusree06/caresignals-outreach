# CareSignals — Healthcare Patient Outreach Intelligence System

> AI-powered patient engagement platform for personalized health communications, risk stratification, and outreach optimization.

---

## Overview

CareSignals is an AI-driven digital engagement infrastructure designed to deliver timely, relevant health communications to insurance members through two modes:

**Broadcast Outreach** — Population-wide nudges for preventive care (flu shots, annual wellness visits, screenings) with basic targeting by age group and chronic condition.

**Personalized Engagement** — Context-aware, adaptive notifications tailored to an individual member's health context, care plan, medication adherence, and engagement history — generated by a Personalization Agent powered by large language models.

All development is conducted on synthetically generated data. No real patient information or PHI is used.

---

## Features

### Patient Overview
- Population-level KPI dashboard: total members, risk distribution, average medication adherence
- Risk level breakdown by age group, insurance plan, and chronic condition
- Outreach channel performance comparison (SMS, Email, Push Notification, Phone)
- High-risk member table sorted by risk score

### Patient Detail
- Individual member profile with conditions, medications, appointments, and preventive care gaps
- Risk score gauge (0–100) with color-coded risk zones
- Medication adherence tracking with refill due dates
- Full outreach history with open, click, and response tracking

### Analytics
- Medication adherence distribution by risk level
- Days since last visit box plots by risk cohort
- Message type effectiveness scatter plot (open rate vs response rate)
- Preventive care gap analysis by care type
- ER visits vs risk score correlation
- Broadcast campaign performance table

### Outreach Generator (Personalization Agent)
- Select any patient and generate a personalized, context-aware outreach message
- Configurable message type, delivery channel, and tone
- LLM uses the patient's full health profile: conditions, medications, care gaps, and appointment history
- Population Analyst: ask natural language questions about the member population via Text-to-SQL (RAG)
- Suggested questions for quick exploration

### Model Performance
- Comparison of 5 ML classifiers: Logistic Regression, Decision Tree, Random Forest, XGBoost, LightGBM
- Color-coded metrics table (Accuracy, F1-Score, AUC-ROC)
- Feature importance chart for the best-performing model
- Training data distribution charts

## Machine Learning Models

| Model | Accuracy | F1-Score | AUC-ROC |
|---|---|---|---|
| Logistic Regression | 0.748 | 0.703 | 0.721 |
| LightGBM | 0.725 | 0.700 | 0.707 |
| Random Forest | 0.720 | 0.691 | 0.718 |
| XGBoost | 0.713 | 0.688 | 0.715 |
| Decision Tree | 0.703 | 0.686 | 0.661 |

**Target variable:** Missed care flag — patient at risk of missed follow-up, medication non-adherence, or preventive care gap.

**Key features:** medication adherence, days since last visit, number of chronic conditions, ER visit frequency, engagement score, hospitalization history, age group.

## Database Schema

| Table | Records | Description |
|---|---|---|
| `patients` | 3,000 | Demographics, risk scores, adherence, engagement metrics |
| `conditions` | ~7,100 | Chronic condition diagnoses per patient |
| `medications` | ~7,100 | Prescriptions with adherence and refill tracking |
| `appointments` | ~7,400 | Scheduled, completed, and missed appointments |
| `preventive_care` | ~10,300 | Preventive care completion and overdue tracking |
| `outreach_messages` | ~10,500 | Message history with open, click, and response rates |
| `risk_scores` | 3,000 | ML-computed risk scores per patient |
| `broadcast_campaigns` | 4 | Population-wide campaign performance |

## Local Setup

```bash
# Clone the repository
git clone https://github.com/Indhusree06/caresignals-outreach.git
cd caresignals-outreach

# Create virtual environment
python -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Generate synthetic data
python generator/generate_data.py

# Train models
python models/train_models.py

# Add your OpenAI API key
echo "OPENAI_API_KEY=sk-your-key-here" > .env

# Run the dashboard
streamlit run dashboard/app.py
```

## Deployment (Streamlit Cloud)

1. Fork this repository
2. Go to [share.streamlit.io](https://share.streamlit.io) and connect your GitHub account
3. Select `dashboard/app.py` as the main file
4. Add your OpenAI API key in **Settings → Secrets**:
   ```toml
   OPENAI_API_KEY = "sk-your-key-here"
   ```
5. Click Deploy
